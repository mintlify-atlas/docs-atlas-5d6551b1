## Get All Orders

<api method="GET" endpoint="/api/admin/orders" />

Retrieve all orders with populated user and product information. Orders are sorted by creation date (newest first).

### Authentication

Requires Clerk authentication with admin role. All admin routes are protected by `protectRoute` and `adminOnly` middleware.

### Response

<ResponseField name="orders" type="array">
  Array of order objects
  <ResponseField name="orders[]._id" type="string">
    MongoDB ObjectId
  </ResponseField>
  <ResponseField name="orders[].user" type="object">
    Populated user object with name and email
  </ResponseField>
  <ResponseField name="orders[].clerkId" type="string">
    Clerk user ID
  </ResponseField>
  <ResponseField name="orders[].orderItems" type="array">
    Array of order items with populated product details
    <ResponseField name="orderItems[].product" type="object">
      Full product object
    </ResponseField>
    <ResponseField name="orderItems[].name" type="string">
      Product name snapshot
    </ResponseField>
    <ResponseField name="orderItems[].price" type="number">
      Price at time of order
    </ResponseField>
    <ResponseField name="orderItems[].quantity" type="number">
      Quantity ordered
    </ResponseField>
  </ResponseField>
  <ResponseField name="orders[].shippingAddress" type="object">
    Shipping address details
    <ResponseField name="shippingAddress.fullName" type="string">
      Recipient full name
    </ResponseField>
    <ResponseField name="shippingAddress.streetAddress" type="string">
      Street address
    </ResponseField>
    <ResponseField name="shippingAddress.city" type="string">
      City
    </ResponseField>
    <ResponseField name="shippingAddress.phoneNumber" type="string">
      Phone number
    </ResponseField>
  </ResponseField>
  <ResponseField name="orders[].paymentResult" type="object">
    Payment information
    <ResponseField name="paymentResult.id" type="string">
      Payment provider ID (e.g., Stripe payment intent ID)
    </ResponseField>
    <ResponseField name="paymentResult.status" type="string">
      Payment status
    </ResponseField>
  </ResponseField>
  <ResponseField name="orders[].totalPrice" type="number">
    Total order price
  </ResponseField>
  <ResponseField name="orders[].status" type="string">
    Order status (see status enum below)
  </ResponseField>
  <ResponseField name="orders[].paidAt" type="string">
    ISO 8601 timestamp when order was marked as paid
  </ResponseField>
  <ResponseField name="orders[].deliveredAt" type="string">
    ISO 8601 timestamp when order was delivered
  </ResponseField>
  <ResponseField name="orders[].createdAt" type="string">
    ISO 8601 timestamp
  </ResponseField>
  <ResponseField name="orders[].updatedAt" type="string">
    ISO 8601 timestamp
  </ResponseField>
</ResponseField>

## Update Order Status

<api method="PATCH" endpoint="/api/admin/orders/:orderId/status" />

Update the status of an order. Triggers automated email notifications and invoice generation based on the new status.

### Authentication

Requires Clerk authentication with admin role.

### Path Parameters

<ParamField path="orderId" type="string" required>
  MongoDB ObjectId of the order to update
</ParamField>

### Request

<ParamField body="status" type="string" required>
  New order status. Must be one of:
  - `pending` - Order created but not paid
  - `paid` - Payment received (triggers invoice generation)
  - `in_preparation` - Order is being prepared
  - `ready` - Order ready for pickup/delivery
  - `delivered` - Order delivered to customer
  - `canceled` - Order canceled
  - `rejected` - Order rejected
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="order" type="object">
  The updated order object with populated user and product data
</ResponseField>

### Automated Actions

When the status changes, the following automated actions occur:

#### Status: `paid`
- Sets `paidAt` timestamp if not already set
- Generates PDF invoice using `generateInvoicePDF()`
- Generates CSV invoice using `generateInvoiceCSV()`
- Sends invoice emails to customer and admin via `sendInvoiceEmails()`
- Invoice number format: `FV-{YEAR}-{ORDER_ID_SUFFIX}`
- Payment method inferred from `paymentResult.id` prefix:
  - `pi_` = Stripe
  - `transfer_` = Transferencia

#### Status: `delivered`
- Sets `deliveredAt` timestamp if not already set
- Sends status update emails to customer and admin

#### All Other Status Changes
- Sends status update emails to customer (via `sendOrderUpdatedClientEmail()`) and admin (via `sendOrderUpdatedAdminEmail()`)
- Respects user's `emailNotifications` preference

<RequestExample>
```bash cURL
curl -X PATCH https://api.example.com/api/admin/orders/507f1f77bcf86cd799439011/status \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "in_preparation"
  }'
```
</RequestExample>

## Get Dashboard Statistics

<api method="GET" endpoint="/api/admin/stats" />

Retrieve aggregated statistics for the admin dashboard including total revenue, orders, customers, and products.

### Authentication

Requires Clerk authentication with admin role.

### Response

<ResponseField name="totalRevenue" type="number" required>
  Total revenue from all orders (sum of all order `totalPrice` fields)
</ResponseField>

<ResponseField name="totalOrders" type="number" required>
  Total number of orders in the system
</ResponseField>

<ResponseField name="totalCustomers" type="number" required>
  Total number of registered users
</ResponseField>

<ResponseField name="totalProducts" type="number" required>
  Total number of products in the catalog
</ResponseField>

### Example Request

```bash
curl -X GET https://api.donpalitojr.com/api/admin/stats \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### Example Response

```json
{
  "totalRevenue": 15750000,
  "totalOrders": 342,
  "totalCustomers": 156,
  "totalProducts": 48
}
```

<Note>
  The `totalRevenue` is calculated by aggregating the `totalPrice` field from all orders, regardless of order status. This includes pending, paid, and delivered orders.
</Note>

<Tip>
  Use this endpoint to populate dashboard cards and metrics displays in the admin panel.
</Tip>
